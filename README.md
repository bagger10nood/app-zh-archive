# app-zh-archive

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为静态 HTML 资源的集合与分发点。

## 项目简介

本仓库存储了一系列独立的 HTML 页面文件，这些页面可能来自不同的项目或用途，统一归档于此以便于版本管理、历史回溯和直接通过 GitHub Pages 或其他静态托管服务进行发布。

仓库本身不包含复杂的后端逻辑，所有内容均为前端静态资源，适合快速部署或作为个人/团队的小型页面存档库。

## 目录说明

```
.
├── index.html          # 仓库首页（可选，用于展示归档列表或导航）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源，如 CSS、JavaScript、图片等
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── ...                 # 其他配置文件（如 .gitignore, CNAME 等）
```

- `pages/`：每个 HTML 文件应相互独立，避免相互依赖，除非明确说明。
- `assets/`：如果页面需要共用样式或脚本，可统一放置在此目录下。

## 页面归档说明

所有 HTML 页面遵循以下原则：

- 每个页面独立完整，可直接在浏览器中打开运行。
- 页面间不应存在硬编码的交叉链接或依赖（除非在归档说明中注明）。
- 归档的页面可能包含示例内容、工具界面、信息展示页等，用途不限。
- 本仓库不保证所有页面长期可用，但会尽量保持历史版本可追溯。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来添加、更新或修复页面。
- 添加新页面时，请将文件放入 `pages/` 目录，并确保文件名具有辨识度。
- 如果页面依赖外部资源（如 CDN 库），请在页面头部注释中注明来源。
- 不建议在仓库中存放敏感信息或大体积文件（如超过 100MB 的媒体文件）。
- 本仓库遵循 [MIT 许可证](LICENSE)，除非页面内另有声明。

## 如何部署

本仓库可直接启用 GitHub Pages：

1. 进入仓库 Settings → Pages。
2. 选择 `main` 分支作为源，目录选择 `/ (root)`。
3. 保存后即可通过 `https://<username>.github.io/app-zh-archive/` 访问。

如需自定义域名，请在根目录放置 `CNAME` 文件。
